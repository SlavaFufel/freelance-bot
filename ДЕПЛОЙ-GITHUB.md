# Хостинг на GitHub Actions (бесплатно, 24/7)

Бот будет сам запускаться по расписанию в облаке GitHub и слать тебе заказы в Telegram —
твой компьютер при этом выключен. Файл workflow уже готов: `.github/workflows/freelance-bot.yml`.

## Что важно знать

- **hh.ru на GitHub не работает** — у серверов GitHub заграничный IP, hh отвечает 403.
  Поэтому в облаке включены источники, которые работают отовсюду:
  **Telegram-каналы, freelance.ru, kadrof.ru** (строка `ENABLED_SOURCES` в workflow).
  Хочешь hh — его нужно крутить с российского IP (дом/RU-VPS).
- **Секреты (токен, chat_id) в код НЕ попадают** — они хранятся в GitHub Secrets,
  а файл `.env` исключён через `.gitignore`.

## Шаги (15 минут, один раз)

### 1. Аккаунт и репозиторий
1. Зарегистрируйся на https://github.com (если ещё нет).
2. Создай новый репозиторий: **New repository** → имя любое (напр. `freelance-bot`).
   - **Private** — код скрыт. Бесплатно даётся 2000 минут Actions/мес — при интервале
     30 мин этого хватает (см. примечание про минуты ниже).
   - **Public** — минуты Actions безлимитны, но код виден всем (секреты всё равно скрыты).
   Выбирай Private, если не хочешь показывать конфиг/резюме.

### 2. Залить проект на GitHub
В PowerShell из папки проекта:
```powershell
cd C:\Users\Слава\Desktop\freelance_bot
git init
git add .
git commit -m "freelance bot"
git branch -M main
git remote add origin https://github.com/ТВОЙ_ЛОГИН/freelance-bot.git
git push -u origin main
```
> `.env` и `freelance_bot.db` не зальются — они в `.gitignore`. Это правильно: токен
> остаётся только у тебя. Если git не установлен — поставь https://git-scm.com или
> используй GitHub Desktop / загрузку файлов через сайт.

### 3. Добавить секреты
В репозитории: **Settings → Secrets and variables → Actions → New repository secret**.
Создай:
- `TELEGRAM_BOT_TOKEN` — твой токен бота.
- `TELEGRAM_CHAT_ID` — твой chat id (`1594402949`).
- (необязательно) `HH_USER_AGENT` — для hh; на GitHub не используется.

### 4. Включить и проверить
1. Вкладка **Actions** → если попросит — нажми «I understand… enable workflows».
2. Слева выбери workflow **freelance-bot** → справа **Run workflow** → **Run workflow**
   (ручной запуск для проверки).
3. Через ~1 минуту открой запуск, посмотри лог шага *Run one cycle* и **проверь Telegram** —
   должны прийти карточки заказов.

Готово. Дальше бот запускается сам каждые 30 минут.

## Как менять

Любая правка — отредактируй файл и `git add . && git commit -m "..." && git push`.

- **Интервал** — в `.github/workflows/freelance-bot.yml`, строка `cron`. Время в **UTC**.
  Примеры: `"*/30 * * * *"` каждые 30 мин, `"0 * * * *"` каждый час, `"*/15 * * * *"` каждые 15.
- **Источники в облаке** — там же, строка `ENABLED_SOURCES`.
- **Каналы / ключевые слова / резюме** — в `config.yaml` (и `git push`).

## Про бесплатные минуты (для Private-репозитория)

GitHub даёт 2000 минут Actions/мес бесплатно для приватных репозиториев. Один прогон
с кэшем зависимостей ≈ 0.5–1 мин. При интервале 30 мин выходит ~900–1400 мин/мес — в лимит
укладывается. Если упрёшься — поставь интервал 60 мин (`cron: "0 * * * *"`) или сделай
репозиторий публичным (тогда минуты не считаются).

## Логи и отладка
Вкладка **Actions** → выбери запуск → шаг **Run one cycle**. Там видно, сколько заказов
получено и отправлено. Если ошибка — текст будет там же.
