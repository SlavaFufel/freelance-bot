# Гайд: настройка и работа с ботом

Пошагово, для запуска с нуля. Всё делается один раз, потом просто запускаешь.

---

## Часть 1. Что нужно получить (3 значения в файл `.env`)

Бот складывает секреты в файл `.env`. Нужны три вещи: **токен бота**,
**твой chat id** и **User-Agent для hh**. Разберём каждое.

### 1.1. `TELEGRAM_BOT_TOKEN` — токен Telegram-бота

Это «ключ» от бота, через которого тебе будут приходить заказы. Бот — это твой
личный «почтальон», он шлёт сообщения только тебе.

1. Открой Telegram, в поиске найди **@BotFather** (официальный, с галочкой).
2. Нажми **Start**, отправь команду `/newbot`.
3. BotFather спросит **имя** бота (любое, например «Мои заказы»).
4. Потом спросит **username** — должен заканчиваться на `bot` (например `slava_orders_bot`).
   Если занят — придумай другой.
5. В ответ BotFather пришлёт строку вида:
   `123456789:AAErASDFqwerTYxxxxxxxxxxxxxxxxxxxxxx`
   — **это и есть токен.** Скопируй его.
6. Вставь в `.env`:
   ```
   TELEGRAM_BOT_TOKEN=123456789:AAErASDFqwerTYxxxxxxxxxxxxxxxxxxxxxx
   ```

### 1.2. `TELEGRAM_CHAT_ID` — куда слать сообщения (твой ID)

Это число — адрес твоего личного чата с ботом. Без него бот не знает, кому писать.

1. Сначала **найди своего бота** по username (тот, что заканчивается на `bot`) и
   нажми **Start** — иначе бот не имеет права тебе писать.
2. Напиши боту любое сообщение, например «привет».
3. Открой в браузере ссылку (подставь свой токен вместо `<ТОКЕН>`):
   ```
   https://api.telegram.org/bot<ТОКЕН>/getUpdates
   ```
4. В ответе (это текст в браузере) найди кусок `"chat":{"id":123456789,`.
   Число после `"id":` — **твой chat id**.
5. Вставь в `.env`:
   ```
   TELEGRAM_CHAT_ID=123456789
   ```

> Если в ответе пусто (`"result":[]`) — значит ты не написал боту. Отправь боту
> сообщение и обнови ссылку.

### 1.3. `HH_USER_AGENT` — «подпись» для hh.ru

hh.ru требует, чтобы программа представлялась (иначе отвечает «403 Forbidden»).
Это просто строка с названием и твоей почтой — никакой регистрации не нужно.

```
HH_USER_AGENT=FreelanceBot/1.0 (fr3izy9@gmail.com)
```

> Можно оставить как есть. hh.ru может не отвечать с зарубежных IP/VPN — если
> видишь 403, проверь, что сидишь с российского IP.

### Итог — файл `.env`

1. Скопируй `.env.example` в `.env` (в той же папке).
2. Заполни три строки:
   ```
   TELEGRAM_BOT_TOKEN=твой_токен
   TELEGRAM_CHAT_ID=твой_id
   HH_USER_AGENT=FreelanceBot/1.0 (fr3izy9@gmail.com)
   ```
   Остальные строки (`TELEGRAM_API_ID`/`HASH`) можно не трогать — они только для
   закрытых каналов через Telethon.

## Часть 2. Установка (один раз)

В PowerShell:

```powershell
cd C:\Users\Слава\Desktop\freelance_bot
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
```

Если `Activate.ps1` ругается на политику запуска скриптов — выполни один раз:
```powershell
Set-ExecutionPolicy -Scope CurrentUser RemoteSigned
```

## Часть 3. Как запускать

```powershell
# 1) Проверка БЕЗ Telegram — заказы печатаются в консоль (токен не нужен):
python run.py --once --dry-run

# 2) Один прогон с отправкой в Telegram:
python run.py --once

# 3) Постоянная работа — опрос каждые 5 минут (Ctrl+C чтобы остановить):
python run.py
```

Начни с варианта 1 (`--dry-run`) — увидишь, какие заказы находятся и какие отклики
генерируются, без спама в Telegram. Когда настроишь ключевые слова и резюме —
переходи на вариант 2, затем 3.

## Часть 4. Как с этим работать каждый день

1. Запусти `python run.py` (или поставь на автозапуск — см. ниже).
2. В Telegram тебе приходят карточки: **заголовок, биржа, бюджет, ссылка, готовый
   черновик отклика.**
3. Открываешь ссылку, проверяешь заказ.
4. Если подходит — **копируешь черновик, правишь под себя и отправляешь сам** на бирже.
   (Бот специально не отправляет автоматически — так аккаунты не банят.)
5. Один и тот же заказ дважды не придёт — бот помнит отправленное в базе `freelance_bot.db`.

> Хочешь «забыть» историю и получить заказы заново (например, после смены настроек) —
> удали файл `freelance_bot.db`, он создастся заново.

## Часть 5. Настройка под себя (`config.yaml`)

Открой `config.yaml` — там всё с комментариями. Самое важное:

- **`responder.portfolio_link`** — вставь ссылку на своё портфолио (соберём вместе).
- **`responder.experience`** — тексты опыта по категориям (уже заполнены, подправь под себя).
  Полное резюме — в файле `РЕЗЮМЕ.md`.
- **`sources.telegram_channels.channels`** — список каналов с заказами.
  ⚠️ **Это ключевое для качества!** Чем точнее каналы (про веб-разработку, ботов,
  Python), тем больше релевантных заказов. Добавляй юзернеймы публичных каналов
  (без `@`), которые читаешь сам.
- **`match.include_keywords`** — что ищем (сайты, боты, парсеры…). Добавляй/убирай слова.
- **`match.exclude_keywords`** — что отсекаем (дизайн, SMM, отзывы…).
- **`match.require_keyword_in_title`** — `true` (точно, но меньше заказов) /
  `false` (больше заказов, но больше мусора). Если приходит слишком мало — поставь `false`.
- **`match.min_score`** — порог. Меньше число = больше заказов (и мусора), больше = строже.

После правок просто перезапусти бота.

## Часть 7. (Опционально) Автозапуск при включении ПК

Чтобы бот работал в фоне постоянно — добавь задачу в «Планировщик заданий» Windows:
действие «Запуск программы», программа — путь к `.venv\Scripts\python.exe`,
аргумент — `run.py`, «Начать в» — папка проекта. Если нужно — помогу настроить.
