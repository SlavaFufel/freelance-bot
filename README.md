# Freelance Bot

Бот мониторит русскоязычные фриланс-/job-площадки, находит подходящие заказы
(**сайты, веб-приложения, Telegram-боты, парсеры, автоматизация**; приоритет — веб),
формирует **черновик отклика под нужды заказчика** и присылает карточку
(описание + ссылка + готовый отклик) в **Telegram**.

> 📖 **Новичку — начни с [ГАЙД.md](ГАЙД.md):** где взять токен бота, chat id, как
> запустить и работать. Резюме для отправки заказчикам — в [РЕЗЮМЕ.md](РЕЗЮМЕ.md).
> Хостинг 24/7 бесплатно (GitHub Actions) — [ДЕПЛОЙ-GITHUB.md](ДЕПЛОЙ-GITHUB.md).

> ⚠️ Бот **не отправляет отклики сам** — только готовит черновик. Отправляешь ты,
> вручную, по ссылке. Это защищает аккаунты на биржах от блокировок.

## Источники

| Источник | Статус | Способ |
|---|---|---|
| **Telegram-каналы** | ✅ проверено | фриланс-каналы через `t.me/s/<канал>` — главный источник |
| **freelance.ru** | ✅ проверено | статический HTML, страница `/task` |
| **kadrof.ru** | ✅ проверено | статический HTML, страница `/work` |
| **Kwork / Weblancer** | 🟡 нужен Playwright | список грузится через JS |
| **FL.ru / YouDo** | 🟡 best-effort (Playwright) | тяжёлый анти-бот; надёжнее локально с РФ-IP |
| **Profi.ru / Яндекс Услуги** | 🔴 выключены | Playwright + сессия исполнителя |
| **hh.ru** | ⚪ выключен | это вакансии на ПОСТОЯННУЮ работу + блок по IP вне РФ |
| ~~Habr Freelance~~ | ❌ закрыт навсегда | сервис прекратил работу (HTTP 410) — удалён |

Фокус — **разовые фриланс-заказы** (сайты, доработки, боты, парсеры, автоматизация, Unity),
а не трудоустройство: вакансии на постоянку отсекаются (`exclude_keywords` + выключенный hh).
Бот **многопользовательский**: друзья получают доступ по ключу (`ACCESS_KEY`).

## Настройка

1. Скопируй `.env.example` → `.env` и заполни:
   - `TELEGRAM_BOT_TOKEN` — токен от [@BotFather](https://t.me/BotFather).
   - `TELEGRAM_CHAT_ID` — твой chat id (напиши боту, открой
     `https://api.telegram.org/bot<ТОКЕН>/getUpdates`, возьми `chat.id`).
   - `HH_USER_AGENT` — осмысленный UA с контактом (иначе hh отдаёт 403).
2. Открой `config.yaml` и подгони под себя:
   - `match.*` — ключевые слова (include / boost / exclude) и `min_score`.
   - `responder.*` — приветствие, ссылку на **портфолио**, тексты опыта по категориям.
   - `sources.telegram_channels.channels` — реальные каналы с заказами (примеры в файле — замени).
   - `enabled_sources` — какие источники включить.

## Запуск

```powershell
python run.py --once --dry-run   # один прогон, вывод в консоль (без Telegram) — для проверки
python run.py --once             # один прогон с отправкой в Telegram
python run.py                    # бесконечный цикл (опрос по poll_interval_sec)
```

## Как это работает

```
Source.fetch() → Order → dedup(SQLite) → Matcher(score) → Responder(шаблон) → Notifier(Telegram)
```

- **Matcher** (`bot/matcher.py`) — скоринг по ключевым словам; веб-разработка имеет повышенный вес.
  По умолчанию требует ключевое слово в заголовке (`require_keyword_in_title`) — это резко повышает точность.
- **Responder** (`bot/responder.py` + `templates/`) — определяет категорию заказа
  (лендинг / магазин / доработка / веб-приложение / бот / автоматизация), детектит стек и подставляет в шаблон.
- **Storage** (`bot/storage.py`) — SQLite `freelance_bot.db`, чтобы не слать дубли.
  Удали этот файл, чтобы «забыть» обработанные заказы (например, после смены `min_score`).

## Ограничения и заметки

- Парсятся только публичные данные; запросы с паузами и ретраями.
- **Kwork и Weblancer** рендерят список заказов через JavaScript — статикой их не взять.
  Чтобы они заработали, поставь Playwright (`pip install playwright && python -m playwright install chromium`);
  источники сами переключатся на headless-браузер. Селекторы могут потребовать сверки при смене вёрстки.
- Telegram-каналы можно проверить вручную: `python -m bot.sources.telegram_channels <канал>`.
- Profi.ru / Яндекс Услуги показывают заказы в основном под логином исполнителя —
  для них нужен Playwright и сохранённая сессия (см. комментарии в `bot/sources/profi.py`).
